AI mate 추천 상품 설정

부족 영양소별로 POS AI mate 상담 화면에 노출될 우리약국 추천 상품을 관리하는 약사웹 화면. 목록(AIM-P-01)에서 부족 영양소 행을 클릭하면 상세 페이지(AIM-P-02)로 전환되고, 상세 하단의 [＋ 상품 추가]로 굿팜 마스터 상품 검색 모달(AIM-P-03)을 연다. 굿팜 추천 상품은 슈퍼어드민이 관리하므로 여기서는 조회만 가능하고, 우리약국 상품은 부족 영양소별 최대 3개까지 등록·정렬·삭제한다.

AIM-P-01추천 상품 설정 — 목록
플랫폼WEB · 약사웹(굿팜 솔루션)화면 IDAIM-P-01
화면 경로약사웹 > AI mate > 추천 상품 설정권한약사웹 관리자 = 자약국 우리약국 상품만 수정/삭제
화면 설명조제 성분코드→DIND 매칭으로 도출된 부족 영양소를 한 행씩 나열. 각 행은 부족 영양소·관련 카테고리·관련 약물 수·우리약국 추천 상품·굿팜 추천 상품·AI mate 노출 상태·최종 수정 컬럼으로 구성된다. 노출 중지(OFF) 행도 목록에서 숨기지 않고 노출 중지 배지로 표시한다. 행 클릭 시 상세 페이지로 전환(모달 아님).
▶ 실제 프로토타입(pharmacy-web.html) · 파라미터 없음 = 전체 목록(부족 영양소 6행). 행을 클릭하면 iframe 내부에서 상세로 전환됩니다.
디스크립션 (영역·기능·인터랙션)
#영역/항목기능 및 정책인터랙션 / 처리·예외
1목록 컬럼부족 영양소 · 관련 카테고리 · 관련 약물 수 · 우리약국 추천 상품 · 굿팜 추천 상품 · AI mate 노출 상태 · 최종 수정.상품 표기는 상품명(+제조사)(매입처별 바코드 상이). 관련 약물 수는 해당 영양소를 부족하게 만드는 성분 개수.
2AI mate 노출 상태ON = AI mate 노출(그린 배지) / OFF = AI mate 노출 중지(옐로 배지). 상태 옆 ⓘ 툴팁으로 의미 안내.OFF 행도 목록에서 숨기지 않음. 노출 여부는 굿팜 정책(슈퍼어드민)이 결정.
3우리약국 추천 상품 셀등록된 우리약국 상품명을 줄바꿈으로 나열. 미설정 시 미설정 표기.목록에는 순위 번호·제조사를 표기하지 않음(상세에서 확인).
4행 클릭행 전체가 클릭 영역. 클릭 시 상세 페이지로 전환(AIM-P-02).모달이 아닌 페이지 전환. 뒤로가기로 목록 복귀.
5목록 정렬기본 정렬 ① AI mate 노출 ON 우선 → ② 최종 수정일 최신 → ③ 관련 약물 수 많은 순.진입 시 자동 정렬. 컬럼 클릭 정렬 여부는 미정.
필수 화면 상태
상태처리 / 화면 문구
노출 ON 행AI mate 노출(그린). 상세에서 편집 가능.
노출 OFF 행AI mate 노출 중지(옐로). 숨기지 않음. 상세에서 사전 등록·수정 가능.
우리약국 미설정우리약국 셀 미설정. 상세에서 추가 유도.
굿팜만 존재굿팜 셀만 채워지고 우리약국 미설정.
목록 0건데이터가 없습니다. 안내만 표기.
AIM-P-02추천 상품 설정 — 상세(페이지)
플랫폼WEB · 약사웹(굿팜 솔루션)화면 IDAIM-P-02
화면 경로AIM-P-01 목록 > 부족 영양소 행 클릭권한우리약국 상품만 등록/수정/삭제/정렬 · 굿팜은 조회만
화면 설명상단 뒤로가기+영양소명, 저장 버튼은 우측 하단. 기본정보(부족 영양소·관련 카테고리·관련 약물 수) → 관련 약물(조회 전용: 성분명·부족해지는 이유·근거) → 추천 상품 설정(우리약국 최대 3개 등록·정렬·삭제, 하단 [＋ 상품 추가] → 그 아래 굿팜 추천 상품 조회) 순으로 구성.
▶ pharmacy-web.html?open=2 · 코엔자임Q10(우리약국 2개 등록·노출 ON, 편집 정상 상태). iframe 내부에서 정렬·삭제·[＋ 상품 추가]를 실제로 조작할 수 있습니다.
디스크립션 (영역·기능·인터랙션)
#영역/항목기능 및 정책인터랙션 / 처리·예외
1상단 헤더← 뒤로가기 + 부족 영양소명. 노출 상태 배지 동반.뒤로가기 → 목록(AIM-P-01) 복귀. OFF면 상단 안내 배너 노출.
2기본정보부족 영양소 · 관련 카테고리 · 관련 약물 수 표기.조회 전용(슈퍼어드민/성분 관리 산출값).
3관련 약물이 영양소를 부족하게 만드는 관련 약물을 주성분 단위로 그룹핑 조회(연결 성분코드가 많을 때 대응 — 예: B12 650코드/126주성분). 성분명 검색 · 한 페이지 10개 그룹 페이지네이션. 그룹 펼침 → 개별 성분코드(제형·함량·이유·근거) 조회.조회/참고용만(편집은 슈퍼어드민). soft delete된 성분은 미노출.
4우리약국 추천 상품부족 영양소별 최대 3개 등록. 각 항목 = 순위 번호(1·2·3) + 상품명 + 제조사 표기(2개 이상이면 ▲▼ 순서 조정), 삭제 가능.3개 도달 시 [＋ 상품 추가] 차단·안내. 변경은 저장 시 확정.
5[＋ 상품 추가]굿팜 마스터 상품 검색 모달(AIM-P-03) 오픈.최대 3개 도달 시에만 숨김. 노출 OFF 상태에서도 추가 가능(사전 세팅).
6굿팜 추천 상품추가 버튼 아래에 굿팜 추천 상품 조회 영역(편집 불가 · 슈퍼어드민 관리).하단 안내: 우리약국 설정 상품이 굿팜(기본) 추천보다 우선 노출. 우리약국 미설정 시 POS 상담에는 굿팜 추천만 노출.
7저장우측 하단 [저장]. 우리약국 상품 추가/수정/삭제/정렬 반영.이력 기록: 수정일시·수정자·항목·전/후. 성공/실패 토스트.
필수 화면 상태
상태처리 / 화면 문구
편집 정상(ON)등록·정렬·삭제·추가 모두 활성. 저장 가능.
완전 미설정우리약국 0개 → [＋ 상품 추가]로 등록 유도. 굿팜만 조회.
노출 OFF상단 안내 배너 + 등록·수정·삭제 허용(사전 세팅). 저장값은 POS 미노출, 노출 재개 시 그대로 사용.
최대 3개 도달[＋ 상품 추가] 비활성 + 안내. 삭제 후 재추가 가능.
삭제된 성분 포함해당 성분/약물 정보 미노출(soft delete).
AIM-P-03굿팜 마스터 상품 검색 — 모달
플랫폼WEB · 약사웹(굿팜 솔루션)화면 IDAIM-P-03
화면 경로AIM-P-02 상세 > [＋ 상품 추가]권한굿팜 마스터 상품 DB만 검색(약국 로컬/POS 아님)
화면 설명우리약국 추천 상품을 추가하기 위한 검색 모달. 굿팜 마스터 상품 DB에서만 검색하며(약국 로컬 재고/POS 아님), 검색 결과에서 선택하면 상세의 우리약국 목록에 추가된다. 중복·결과 0·최대 3개 도달을 각각 안내한다.
▶ pharmacy-web.html?open=2&modal=search · 상세 진입 후 검색 모달 자동 오픈. iframe 내부에서 검색어 입력·선택을 실제로 조작할 수 있습니다.
디스크립션 (영역·기능·인터랙션)
#영역/항목기능 및 정책인터랙션 / 처리·예외
1검색 입력상품명으로 검색. 굿팜 마스터 상품 DB만 대상(결과 표기는 상품명(+제조사)). 바코드는 매입처별 상이로 검색·표기 미사용.약국 로컬/POS 재고는 검색되지 않음.
2검색 결과상품명(+제조사) 리스트. 선택 시 우리약국 목록에 추가.결과 없음 → 검색 결과가 없습니다.
3중복 방지이미 등록된 상품 재선택 차단.안내 문구 이미 추가된 상품입니다.
4최대 3개 제한부족 영양소별 우리약국 최대 3개.도달 시 추가 차단(버튼 비활성)·안내.

검색 결과 0검색 결과가 없습니다.
중복 추가 시도이미 추가된 상품입니다.
최대 3개 도달추가 차단·안내(모달 진입 자체 또는 선택 시).
CASES약사웹 · 케이스별 화면— 노출 상태·등록 상태에 따라
Case 1목록 — 노출 중지(OFF) 행OFF
▶ pharmacy-web.html?case=off-list · 비타민 D(노출 중지) 행만 표시
  • 달라지는 점 AI mate 노출 상태 컬럼에 AI mate 노출 중지(옐로) 배지. OFF여도 목록에서 숨기지 않음.
  • 정책 근거 OFF 상태는 목록에 노출 중지로 표시하고, 상세는 OFF에서도 편집(사전 세팅) 허용.
  • 인터랙션 행 클릭 시 OFF 상세(Case 6, open=4)로 전환 — 편집 가능.
Case 2목록 — 완전 미설정 행미설정
▶ pharmacy-web.html?case=empty · 마그네슘(완전 미설정) 행만 표시
  • 달라지는 점 굿팜·우리약국 추천 상품 모두 미설정. 상담에서 이 영양소는 추천 상품 없이 안내됨.
  • 정책 근거 우리약국은 약국이 직접 등록해야 하며, 미등록 시 상세에서 추가를 유도.
  • 인터랙션 행 클릭 → 상세(Case 5, open=3) 진입 후 [＋ 상품 추가]로 등록 유도.
Case 3목록 — 굿팜만 존재(우리약국 미설정)참고
▶ pharmacy-web.html?case=gp-only · 비타민 B12(굿팜만) 행만 표시
  • 달라지는 점 굿팜 추천 상품 셀만 채워지고 우리약국은 미설정. 상담에는 굿팜 추천만 노출.
  • 정책 근거 굿팜 추천 상품은 슈퍼어드민이 관리하고 약사웹은 조회만 가능.
  • 인터랙션 상세에서 굿팜은 조회 영역, 우리약국은 [＋ 상품 추가]로 별도 등록.
Case 4목록 — 우리약국 등록됨(정상)ON
▶ pharmacy-web.html?case=store-set · 코엔자임Q10(우리약국 2개 등록) 행만 표시
  • 달라지는 점 우리약국 추천 상품 셀에 등록 상품명 나열. 노출 ON 상태.
  • 정책 근거 부족 영양소별 우리약국 최대 3개. 상담에서 우리약국+굿팜 함께 노출.
  • 인터랙션 행 클릭 → 편집 정상 상세(AIM-P-02, open=2)로 전환.
Case 5상세 — 완전 미설정 → 추가 유도미설정
▶ pharmacy-web.html?open=3 · 마그네슘(우리약국 0개·굿팜도 미설정)
  • 달라지는 점 우리약국 추천 상품 목록이 비어 있고 [＋ 상품 추가]로 등록을 유도. 굿팜 조회 영역도 비어 있음.
  • 정책 근거 미등록 부족 영양소는 상세에서 검색 모달(AIM-P-03)로 최초 등록.
  • 인터랙션 [＋ 상품 추가] → 검색 모달 → 선택 → 목록 추가 → 저장.
Case 6상세 — 노출 중지(OFF) · 사전 등록 허용OFF
▶ pharmacy-web.html?open=4 · 비타민 D(노출 중지) — 상단 안내 배너 · 편집은 허용(사전 세팅)
  • 달라지는 점 상단 안내 배너 “현재 굿팜 정책에 따라 이 부족 영양소의 POS AI mate 추천 노출이 중지되었습니다. 다만 우리약국 추천 상품은 지금도 등록·수정할 수 있으며, 노출이 다시 시작되면 저장한 설정값이 그대로 사용됩니다.” 하단 라벨 지금 등록·수정한 설정값은 노출 재개 시 그대로 사용(POS에는 아직 미노출). 등록/수정/삭제/정렬·저장 모두 활성.
  • 정책 근거 OFF는 POS 노출 중지일 뿐 편집 차단이 아님. 노출 재개 전 미리 세팅할 수 있게 편집 허용(Q13 확정), 저장값은 재개 시 그대로 사용.
  • 인터랙션 저장 시 설정값만 반영되고 POS에는 노출되지 않음. 슈퍼어드민이 노출 ON으로 바꾸면 저장된 값이 즉시 사용됨.
Case 7검색 모달 — 굿팜 마스터 상품 검색모달
▶ pharmacy-web.html?open=2&modal=search · 상세 진입 후 검색 모달 자동 오픈
  • 달라지는 점 상세 위에 검색 모달 오버레이. 굿팜 마스터 상품 DB에서만 검색(약국 로컬/POS 아님), 선택 시 우리약국 목록에 추가.
  • 정책 근거 AIM-P-03 — 검색 대상은 굿팜 마스터 상품 DB으로 한정. 중복·결과0·최대3 도달을 각각 안내.
  • 인터랙션 항목 선택 → 목록 추가 → 모달 닫힘 → 상세에서 저장으로 확정.
Case 8데이터 케이스 — 등록·검색 예외 문구문구
  • 최대 3개 도달 우리약국 추천 상품이 부족 영양소별 3개에 도달하면 [＋ 상품 추가] 비활성·안내. 삭제로 자리를 비우면 재추가 가능.
  • 중복 추가 이미 등록된 상품을 다시 선택하면 “이미 추가된 상품입니다.” 안내, 추가되지 않음.
  • 검색 결과 0 굿팜 마스터 상품 DB에서 일치 항목이 없으면 “검색 결과가 없습니다.” 표시.
  • 삭제된 성분 포함 soft delete된 성분코드의 관련 성분/약물 정보는 약사웹에서 미노출. 나머지 관련 약물은 정상 표시.
  • 저장 성공/실패 우리약국 상품 추가/수정/삭제/정렬 저장 시 성공/실패 토스트. 성공 시 이력 기록(수정일시·수정자·항목·전/후).
  • 확인 필요 최대 도달·중복·결과0의 정확한 토스트/배너 위치와 저장 실패 재시도 UX는 프로토타입 기준 문구를 인용하되 세부 확정은 TBU.
