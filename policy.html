1. 데이터 모델 · 관리 기준

관리·편집 기준(grain)을 3층으로 분리한다. 성분↔영양소가 M:N이므로, 각 정보가 어느 축에 종속되는지에 따라 편집 위치가 갈린다.

정보관리 기준편집 화면
추천 상품 · 관련 카테고리 · 추천 효능군 · 노출 상태부족 영양소슈퍼어드민 AI mate 추천 관리
성분명 · 제형성분코드(공통값)슈퍼어드민 AI mate 성분 관리
부족해지는 이유 · 근거/참고문헌성분코드 × 부족 영양소(교차)성분 관리 → 연결 부족 영양소 목록
  • 성분명은 성분코드 기준 공통값 → 한 번 수정하면 연결된 모든 부족 영양소·AI mate에 공통 반영.
  • 이유·근거는 (성분코드 × 부족 영양소) 조합 고유값 → 조합별 독립(같은 성분이어도 영양소 다르면 내용 다름).

2. 성분코드 병합 정책 (함량/제형 차이 유지)

  • 성분코드 앞자리만으로 단순 병합하지 않는다.
  • DIND 성분코드는 성분 + 함량 + 제형에 따라 세분화되며, 함량/제형에 따라 부족 영양소·이유·근거가 달라질 수 있다.
  • 따라서 상세 성분코드의 함량/제형 차이를 무시하고 근거를 병합하지 않는다.
  • "같은 성분" 범위(함량 포함 여부)는 정책 확정 필요.

POS 조회 흐름

  1. 처방약 성분코드 수신 (POS/DB에서 함께 전달 — 전제)
  2. DIND 상세 성분코드 매칭
  3. 상세 성분코드 → 부족 영양소 조회
  4. 부족 영양소 기준 추천 상품 조회(굿팜/우리약국)
  5. (상세 성분코드 × 부족 영양소) 기준 부족해지는 이유·근거 문구 조회
핵심: 추천 상품은 부족 영양소에 붙인다. 단, 이유·참고문헌은 상세 성분코드 기준으로 유지하고 함량/제형 차이를 무시해 병합하지 않는다.

3. DIND 원천 vs 운영 수정(override) · 재업로드

상세 정책 필요 (4·5번)override 저장 구조·재업로드 충돌 처리 상세 설계 필요. 현재 슈퍼어드민에는 DIND 신규 추가·일괄 업로드 기능이 없음 → 재업로드 UX는 이 기능 부활 여부와 함께 확정.

기본 원칙

  • DIND 원천값은 보존한다.
  • 운영자 수정값은 override로 별도 저장한다.
  • 노출 시 override 있으면 우선, 없으면 DIND 원천값 사용.

수동 수정 가능 항목

부족해지는 이유 · 근거/참고문헌 · 관련 카테고리 · 추천 효능군 · 굿팜 추천 상품 · 노출 상태.

재업로드 시 처리

  • 재업로드해도 override는 자동 삭제하지 않는다.
  • 원천이 바뀌면 원천 변경 있음 상태 표시 → 운영자가 override 유지/해제 판단.
상태의미
원천값 사용override 없음 — DIND 원천 그대로
운영 수정값 사용override 우선 노출(운영 수정 적용 중 배지)
원천 변경 있음override 있는 항목에서 DIND 원천이 바뀜 — 판단 대기
검토 필요결측·표현/근거 리스크 등 노출 전 검수 대상

4. 영양소명 정규화

DIND 표기값을 그대로 쓰지 않고 굿팜 표준 영양소명으로 정규화한다. (DIND에 Vit-B12/비타민 B12/cobalamin 등 혼재 → 같은 영양소가 분리되면 관리가 꼬임)

매핑: DIND 표기값 → 굿팜 표준 영양소명 → 부족 영양소 Master

5. 추천 상품 연결 (수동 큐레이션)

  • 굿팜 추천 상품: 슈퍼어드민 부족 영양소별 최대 2개.
  • 우리약국 추천 상품: 약사웹 부족 영양소별 최대 3개.
  • 관련 카테고리·추천 효능군은 MVP에서 자동 매칭 Key 아님 — 표시/운영 참고값.
  • 후순위: 고도화 시 카테고리/효능군 기반 상품 후보 자동 추천 검토.

6. 복합제 처리

  • 복합제는 성분별로 분해해 매칭하고(예: 세비카정 → 성분 A/B), 각 성분의 부족 영양소를 합산한다.
  • 동일 영양소가 여러 성분에서 중복되면 POS에서 1회만 노출, 단 근거에는 관련 성분을 함께 표시한다.
  • UI: 하나의 영양소 블록 + 내부 관련 성분.
예 — 부족 영양소: 마그네슘 / 관련 성분: 성분 A, 성분 B / 근거: · 성분 A 기준 · 성분 B 기준

8. 삭제 정책 재정리

정책 확정 필요삭제 액션 제공 여부·조건은 논의 필요. 아래는 현재 제안(프로토타입 반영).
  • 부족 영양소 Master 삭제는 기본 액션으로 제공하지 않는다. 운영 제외는 AI mate 노출 OFF.
  • 삭제 가능(모두 충족 시): 노출 OFF · 약국 설정 상품 0건 · 우리약국 추천 상품 0건.
  • 삭제 불가: 노출 ON · 약국 설정 1건 이상 · 운영 중 → 삭제 버튼 비활성화 + 사유 툴팁.
  • 권장: MVP에서 강제 삭제 없음, 삭제보다 OFF를 기본으로.

약국 설정 수 (참고 정보) — 논의 필요

논의 필요삭제 기능 제거 기준에서는 약국 설정 수를 삭제 판단용 필수값으로 쓰지 않는다. 다만 운영 현황 파악을 위해 상세 화면에 참고 정보로 표시한다.
  • 표시 예: 우리약국 추천 상품을 등록한 약국 수: N곳
  • 삭제 게이트가 아니라 운영 참고 지표로만 사용.

9. 약사웹 OFF 상태 정책

슈퍼어드민에서 OFF로 변경해도 약사웹에서 완전히 숨기지 않는다.

핵심: OFF는 데이터 삭제가 아니라 POS 추천 노출 중지 상태. 약사가 "설정이 사라졌다" 오해하지 않도록 노출 중지 상태로 보여준다.

10. 화면별 반영 (현재 프로토타입)

  • 슈퍼어드민 · AI mate 추천 관리(부족 영양소): 목록(카테고리·관련 약물 수·노출·수정일) → 상세 페이지(노출 토글 상단·카테고리·굿팜 상품 최대 2·관련 약물·수정 이력, 저장/삭제 우측 하단).
  • 슈퍼어드민 · AI mate 성분 관리(성분코드): 목록(성분코드·성분명·부족 영양소 개수) → 상세 페이지(성분명 공통값 수정·연결 부족 영양소별 이유/근거 수정·연결 추가/삭제).
  • 약사웹(GP 디자인): 부족 영양소별 우리약국 상품 설정, 상세 페이지, ON/OFF 정책, 바코드→제조사.
  • POS(시안 D): 좌(근거)/우(우리약국·굿팜 2열), 상품 있는 영양소만 노출, 10개+더보기, 우리약국 3개 우선순위.
