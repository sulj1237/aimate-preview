1. 개요

  • 대상: POS AI mate · 약사웹 · 슈퍼어드민 (내부 운영 화면).
  • 표현 목적: 추천/문구는 상담 참고로 작성한다(치료·예방·효능 단정 표현 금지).
  • 전제: 처방약 수신 시 성분코드를 함께 받는다. 본 정책 범위는 성분코드 이후의 DIND 매칭·관리·노출이다.

2. 확정 정책 요약

#영역확정 기준
1매칭성분코드가 완전히 같을 때만 매칭
2데이터 관리정보별 관리 단위 분리 — 추천 상품·노출은 영양소별, 성분명은 성분별, 이유·근거는 (성분×영양소)별
3추천 상품부족 영양소별 굿팜 최대 2 · 우리약국 최대 3, 굿팜 마스터 상품 DB에서 선택
4운영 수정DIND 원본은 보존하고, 운영자가 고친 값이 있으면 그 값을 우선 노출
5정규화DIND 표기 → 굿팜 표준 영양소명 기준 노출
6복합제완제품 성분코드로 매칭해 DIND 제공 부족 영양소 사용 (원인 성분 표시·1회 합치기는 추후 검토)
7결측/노출추천 상품이 있으면 노출, 없는 항목만 생략
8OFFPOS 노출만 중지(데이터 보존 · 약사웹 편집 유지)
9삭제삭제 항목만 전 서비스 미노출 + 원천·이력 보존(soft delete)

3. 데이터 기준

정보마다 관리·편집하는 단위가 다르다. 한 성분이 여러 영양소에, 한 영양소가 여러 성분에 걸치기 때문에, 어떤 정보를 어느 단위로 관리할지 아래처럼 나눈다.

관리 단위이 단위로 관리하는 정보편집 주체
부족 영양소 단위추천 상품(굿팜/우리약국) · 관련 카테고리 · 노출 상태슈퍼어드민 · 약사웹(우리약국만)
성분(성분코드) 단위성분명 등 성분 공통 정보슈퍼어드민
(성분 × 영양소) 단위부족해지는 이유 · 근거(참고문헌)슈퍼어드민
  • 성분명은 성분 단위 공통값이라, 한 번 고치면 그 성분과 연결된 모든 영양소·화면에 함께 반영된다.
  • 이유·근거는 (성분 × 영양소) 조합마다 다르다. 같은 성분이어도 영양소가 다르면 이유·근거가 따로 관리된다.
  • 성분명이 같아도 성분코드가 다르면 서로 다른 성분으로 관리한다.

관련 성분 목록 표시 (대량 대응)

부족 영양소를 유발하는 것은 성분이며, DIND는 성분코드(주성분+함량+제형) 단위다. 한 영양소에 성분코드가 많을 수 있다(예: 비타민 B12 = 650 성분코드).

  • 관련 성분 목록은 성분코드 단위 리스트로 표시한다. 제형 표기 형식 = 한글명(코드)(예: 정제, 저작정(TB)).
  • 상단에 총 개수를 표기하고, 페이지네이션(한 페이지 10개 · 10개 이하도 UI 표시) 으로 탐색한다.
  • 슈퍼어드민(관리): 컬럼 = 성분코드 · 성분명 · 제형 · 부족해지는 이유 · 근거(참고문헌). 성분명·코드 검색 · 레코드(행) 클릭 시 성분 관리로 이동(수정).
  • 약사웹(조회): 컬럼 = 성분명 · 제형 · 부족해지는 이유 · 근거(참고문헌). 성분명 검색 제공(성분코드 열·이동 없음). 편집은 슈퍼어드민.
  • 매칭·수정·삭제·이유/근거 관리는 성분코드(9자리 전체) 기준으로 유지한다.

4. 매칭/노출 정책

매칭

  • DIND 매칭은 성분코드가 완전히 같을 때만 수행한다. 매칭·관리 단위는 전체 성분코드다.
  • 근거: 성분코드 앞자리가 같아도 함량·제형에 따라 부족 영양소·이유·근거가 달라질 수 있어, 전체 일치 기준이어야 처방에 맞는 근거만 노출된다.

추천 상품

항목기준
굿팜 추천 상품부족 영양소별 최대 2개
우리약국 추천 상품부족 영양소별 최대 3개
검색·선택 대상굿팜 마스터 상품 DB
검색 키상품명(바코드는 매입처별 상이로 검색·표기 모두 미사용)
약사웹 표기상품명(+제조사) 중심
관련 카테고리표시·운영 참고값

5. 운영 수정/원천 데이터 정책

  • DIND에서 받은 원본 값과 운영자가 고친 수정 값을 따로 저장한다.
  • 화면에 보여줄 때는 수정 값이 있으면 그 값을, 없으면 원본 값을 쓴다.
  • DIND를 다시 올려도 운영자 수정 값은 그대로 유지한다(지울지 여부는 운영자가 판단).
  • 수정 가능 항목: 부족해지는 이유 · 근거(참고문헌) · 관련 카테고리 · 굿팜 추천 상품 · 노출 상태.
  • 원본이 바뀌었을 때 충돌 처리·표시 방식은 개발 상세 설계 항목(→ §10 TBU).

6. 복합제/결측 처리

영양소 정규화

  • DIND 표기값은 굿팜 표준 영양소명으로 정규화해 노출한다. DIND 원문 표기는 필요 시 참고값으로만 사용한다.

복합제

  • 복합제(성분이 여러 개인 약)는 완제품 성분코드로 매칭하며, DIND가 그 코드 단위로 제공하는 부족 영양소·원인 성분·이유·근거를 그대로 사용한다.
  • 같은 부족 영양소가 여러 원인 성분에서 겹칠 때 1회로 합쳐 노출 + 관련 성분 표시하는 처리는 추후 검토.
  • 서로 다른 처방약이 같은 부족 영양소를 유발하면 약마다 각각 노출한다.

결측/노출 (POS 추천 블록)

값 표기 — 무관 = 값에 상관없이 동일 처리 · 없음 = 해당 항목 값이 비어 있음.

추천 상품부족해지는 이유근거(참고문헌)관련 카테고리처리
있음무관무관무관블록 노출
있음없음무관무관노출 + 부족해지는 이유 문구 생략
있음무관없음무관노출 + 근거(참고문헌) ⓘ 아이콘 미노출
있음무관무관없음노출 + 관련 카테고리 배지 미노출
없음무관무관무관블록 미노출
  • 판정 기준: 추천 상품(필수값)이 하나라도 있으면 블록을 노출하고 없는 항목만 생략한다. 추천 상품이 없을 때만 블록을 미노출한다.

7. ON/OFF/삭제 정책

ON/OFF (노출 중지)

  • OFF는 POS AI mate 노출을 중지하는 상태다(데이터 보존).
영역OFF 시 처리
POS AI mate미노출
약사웹 목록노출 중지 상태로 표시
약사웹 편집우리약국 추천 상품 등록·수정·삭제 허용
다시 ON저장된 설정값 그대로 POS 반영

삭제 (해당 항목만 전 서비스 미노출)

  • 삭제하면 그 성분/부족 영양소 항목이 슈퍼어드민·약사웹·POS 전체에서 미노출되고, 원천 데이터·삭제 이력은 보존한다(soft delete). 삭제한 그 항목만 미노출한다.
  • 부족 영양소 삭제: 해당 부족 영양소와 연결된 굿팜 상품·카테고리·관련 성분 매칭까지 함께 미노출.
  • 성분 삭제: 삭제 단위는 원인 성분이다. 삭제한 원인 성분 기준 추천·근거만 미노출하고, 복합제의 남은 원인 성분 기준 추천은 정상 노출한다.
  • 일시 제외는 OFF를 사용한다.
