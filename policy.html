1. 개요

  • 대상: POS AI mate · 약사웹 · 슈퍼어드민. 고객앱 직접 노출은 MVP 제외.
  • 표현 목적: 추천/문구는 상담 참고로 한정한다(치료·예방·효능 단정 표현 금지). 근거·표현 리스크 항목은 노출 OFF로 분리한다.
  • 전제: 처방약 수신 시 성분코드를 함께 받는다. 본 정책 범위는 성분코드 이후의 DIND 매칭·관리·노출이다.

2. 확정 정책 요약

#영역확정 기준
1매칭성분코드 전체 일치 기준. 부분 코드(4/7자리)는 운영 참고·검색 보조로만 사용
2데이터 관리부족 영양소 / 성분코드 / (성분코드 × 부족 영양소) 3층 분리
3추천 상품부족 영양소별 굿팜 최대 2 · 우리약국 최대 3, 굿팜 보유 상품에서 선택
4운영 수정원천값과 운영 수정값 분리, 운영 수정값 우선
5정규화DIND 표기 → 굿팜 표준 영양소명 기준 노출
6복합제성분별 분해 → 영양소 합산 → 복합제 내 중복 1회 노출
7결측/노출추천 상품이 있으면 노출, 없는 항목만 생략
8OFF데이터 삭제 아님 — POS 노출 중지(약사웹 편집 유지)
9삭제물리 삭제 아님 — 전 서비스 미노출 + 원천·이력 보존

3. 데이터 기준

관리·편집 단위를 3층으로 분리한다. 같은 성분이 여러 영양소에, 같은 영양소가 여러 성분에 걸치는 M:N 구조이므로 정보별 종속 축이 다르다.

관리 기준(grain)종속 정보편집 주체
부족 영양소추천 상품(굿팜/우리약국) · 관련 카테고리 · 노출 상태슈퍼어드민 · 약사웹(우리약국만)
성분코드성분명 등 성분 공통 정보슈퍼어드민
성분코드 × 부족 영양소부족해지는 이유 · 근거/참고문헌슈퍼어드민
  • 성분명은 성분코드 공통값 — 한 번 수정하면 연결된 모든 부족 영양소·화면에 공통 반영된다.
  • 이유·근거는 (성분코드 × 부족 영양소) 조합 고유값 — 같은 성분이어도 영양소가 다르면 독립이다.
  • 성분명이 같아도 성분코드가 다르면 별개 레코드로 관리한다.

4. 매칭/노출 정책

매칭

  • DIND 매칭은 성분코드 전체 일치 시 수행하며, 매칭·관리 단위는 전체 성분코드다.
  • 부분 코드(앞 4/7자리) 단위는 운영 참고·검색 보조로만 사용한다(자동 병합 대상 아님).
  • 근거: 앞자리가 같아도 함량·제형에 따라 부족 영양소·이유·근거가 달라질 수 있어, 전체 일치 기준이어야 처방에 맞는 근거만 노출된다.

추천 상품

항목기준
굿팜 추천 상품부족 영양소별 최대 2개
우리약국 추천 상품부족 영양소별 최대 3개
검색·선택 대상굿팜 보유 상품(약국 로컬/POS 재고 아님)
약사웹 표기바코드가 아닌 제조사 중심
관련 카테고리자동 매칭 Key 아님 — 표시·운영 참고값
추천 효능군MVP 화면·정책에서 제외

5. 운영 수정/원천 데이터 정책

  • DIND 원천값과 운영자 수정값(override)을 분리 관리한다.
  • 노출 우선순위: 운영 수정값이 있으면 우선 사용, 없으면 원천값 사용.
  • 재업로드/갱신 시 운영 수정값은 자동 삭제하지 않는다(운영자 판단으로 유지/해제).
  • 수정 가능 항목: 부족해지는 이유 · 근거/참고문헌 · 관련 카테고리 · 굿팜 추천 상품 · 노출 상태.
  • 원천 변경/충돌 상태의 판정·표시 로직은 개발 상세 설계 항목(→ §10).

6. 복합제/결측 처리

영양소 정규화

  • DIND 표기값은 굿팜 표준 영양소명으로 정규화해 노출한다. DIND 원문 표기는 필요 시 참고값으로만 사용한다.

복합제

  • 복합제는 성분별로 분해해 DIND 매칭 후, 성분별 부족 영양소를 합산한다.
  • 복합제 내부 동일 부족 영양소 중복 → 1회만 노출(관련 성분·근거는 함께 표시 가능해야 함).
  • 서로 다른 처방약 간 동일 영양소 → 약물별로 각각 노출한다.

결측/노출 (POS 추천 블록)

추천 상품부족해지는 이유참고문헌관련 카테고리처리
있음무관무관무관블록 노출
있음없음——노출 + 근거 문구만 생략
있음—없음—노출 + 정보 아이콘만 미노출
있음——없음노출 + 카테고리 배지만 미노출
없음———블록 자체 미노출
  • 판정 기준: 추천 상품(필수값)이 하나라도 있으면 블록을 노출하고 없는 항목만 생략한다. 추천 상품이 없을 때만 블록을 미노출한다.

7. ON/OFF/삭제 정책

ON/OFF (노출 중지)

  • OFF는 데이터 삭제가 아니라 POS AI mate 노출 중지 상태다.
영역OFF 시 처리
POS AI mate미노출
약사웹 목록노출 중지 상태로 표시(숨기지 않음)
약사웹 편집우리약국 추천 상품 등록·수정·삭제 허용
다시 ON저장된 설정값 그대로 POS 반영

삭제 (전 서비스 미노출)

  • 삭제는 물리 삭제가 아니라 전 서비스 미노출 처리이며, 원천 데이터와 삭제 이력은 보존한다.
  • 부족 영양소 삭제: 슈퍼어드민·약사웹·POS에서 미노출(연결 굿팜 상품·카테고리·관련 약물 매칭 포함).
  • 성분 삭제: 해당 성분 기준 추천·근거를 전 서비스에서 미노출. 복합제는 삭제되지 않은 다른 성분 기준 추천은 정상 노출.
  • 일시 제외 목적이면 삭제가 아니라 OFF를 사용한다.
  • 약국 설정 수는 삭제 가능 여부 판단값이 아니라 운영 참고 지표다.
