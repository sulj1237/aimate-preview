1. 데이터 모델 · 관리 기준

관리·편집의 기준(grain)을 3층으로 분리한다. 같은 성분이 여러 영양소에 걸치고(예: 오메프라졸 → B12·칼슘·마그네슘), 같은 영양소가 여러 성분에 걸치므로(M:N), 각 정보가 어느 축에 종속되는지에 따라 편집 위치가 갈린다.

  • 성분명은 성분코드 기준 공통값이라 한 번 수정하면 연결된 모든 부족 영양소·AI mate 화면에 공통 반영된다.
  • 이유·근거는 (성분코드 × 부족 영양소) 조합 고유값이라 조합별로 독립이다. 같은 성분이어도 영양소가 다르면 이유/근거가 다르다.

2. 성분코드 매칭 기준 — 전체 일치 (그룹핑·병합 없음)

  • 처방 약물 매칭은 성분코드 전체가 일치하는 경우에만 수행한다. 앞 4자리 등 일부 자리로 그룹핑·병합하지 않는다.
  • 이유: 성분코드 앞 4자리가 같아도 함량·제형에 따라 부족 영양소·부족해지는 이유·근거가 달라질 수 있다. 부분 일치로 묶으면 해당 처방에 맞지 않는 근거가 노출될 위험이 있다.
  • 관리 단위도 전체 성분코드로 한다. 성분명이 같아도 성분코드가 다르면 별개 레코드로 관리하며, 이유·근거는 (성분코드 × 부족 영양소) 조합별로 독립 유지한다.
  • 운영 보조(입력 참고): 슈퍼어드민 DIND 추가 시 입력한 성분코드의 앞 4자리가 같은 기존 성분코드가 있으면 그 성분명·연결 부족 영양소·이유·근거를 참고용으로 함께 표시한다. 어디까지나 입력 참고이며 자동 복사·병합하지 않는다.

POS 조회 흐름

  1. 1처방약 성분코드 수신 (POS/DB에서 함께 전달 — 전제)
  2. 2DIND 상세 성분코드 매칭
  3. 3상세 성분코드 → 부족 영양소 조회
  4. 4부족 영양소 기준 추천 상품 조회(굿팜/우리약국)
  5. 5(상세 성분코드 × 부족 영양소) 기준 부족해지는 이유·근거 문구 조회
핵심: 추천 상품은 부족 영양소에 붙인다. 단, 부족해지는 이유·참고문헌은 상세 성분코드 기준으로 유지하고 함량/제형 차이를 무시해 병합하지 않는다.

3. DIND 원천 vs 운영 수정(override) · 재업로드 병합

DIND 원천 데이터와 운영자 수동 수정값을 분리해서 관리한다.

기획 기준 확정 · 상세 설계는 개발 이관 — 아래 override 대상 항목·노출 우선순위·상태값 4종은 기획 기준으로 확정한다. 실제 저장 스키마와 재업로드 병합 실행 로직은 서버 설계 영역이다. 슈퍼어드민에는 DIND 단건 수동 추가(검토 필요·노출 OFF로 등록)가 제공되며, 엑셀 일괄 업로드는 추후 도입 시 업로드 화면 UX만 재확인한다.

기본 원칙

  • DIND 원천값은 보존한다.
  • 운영자 수정값은 override 값으로 별도 저장한다.
  • AI mate 노출 시 override 값이 있으면 override 우선, 없으면 DIND 원천값을 사용한다.

수동 수정 가능 항목

부족해지는 이유 · 근거/참고문헌 · 관련 카테고리 · 굿팜 추천 상품 · 노출 상태.

재업로드 시 처리

  • DIND 재업로드가 발생해도 운영자 override 값은 자동 삭제하지 않는다.
  • 원천 데이터가 변경된 경우 원천 변경 있음 상태를 표시하고, 운영자가 override 유지/해제 여부를 판단할 수 있게 한다.

상태값

상태의미
원천값 사용override 없음 — DIND 원천값 그대로 노출
운영 수정값 사용override 있음 — 운영 수정값을 우선 노출(운영 수정 적용 중 배지)
원천 변경 있음override가 있는 항목에서 DIND 원천이 바뀜 — 운영자 판단 대기

UI 반영

  • 상세 화면에는 원천값과 운영 수정값을 구분해서 보여준다.
  • 운영 수정값이 적용 중이면 운영 수정 적용 중 배지를 표시한다.

4. 영양소명 정규화

DIND 표기값을 그대로 쓰지 않고 굿팜 표준 영양소명으로 정규화한다.

  • 이유: DIND에는 동일 영양소가 여러 방식(예: Vit-B12 / 비타민 B12 / cobalamin / Ca / 칼슘 / Mg / 마그네슘)으로 표기될 수 있어, 그대로 쓰면 같은 영양소가 여러 개로 분리되어 추천 상품 관리가 꼬인다.
  • 매핑 순서: DIND 표기값 → 굿팜 표준 영양소명 → 부족 영양소 Master.

  • 화면 정책: 운영 화면에는 표준 영양소명만 노출하고, DIND 원문 표기는 상세/관리 참고값으로만 제공한다.

5. 추천 상품 연결 (수동 큐레이션)

MVP에서는 추천 상품을 자동 매칭하지 않고 부족 영양소별 수동 큐레이션으로 운영한다.

  • 굿팜 추천 상품: 슈퍼어드민에서 부족 영양소별 최대 2개 등록.
  • 우리약국 추천 상품: 약사웹에서 부족 영양소별 최대 3개 등록.
  • 상품 검색·표기: 추천 상품은 굿팜 보유 상품에서만 검색해 선택한다(약국 로컬/POS 재고 아님). 검색 키는 상품명 또는 바코드(굿팜 보유 상품 마스터 기준), 화면 표기는 바코드가 아닌 제조사 로 한다(매입처별 바코드가 달라 약국 간 혼동을 줄이기 위함).
  • 관련 카테고리는 MVP에서 자동 상품 매칭 Key로 사용하지 않는다 — 표시 및 운영 참고값으로만 사용한다. (추천 효능군은 전 화면에서 제외)
  • 후순위: 고도화 단계에서 관련 카테고리 기반 상품 후보 자동 추천을 검토한다.

6. 복합제 처리

처방약 하나에 성분이 여러 개인 복합제 케이스를 처리한다.

  • 기본: 복합제는 성분별로 분해해 DIND 매칭을 수행하고(예: 세비카정 → 성분 A / 성분 B), 각 성분별 부족 영양소를 조회한 뒤 합산한다.
  • 중복 처리(복합제 내부): 하나의 복합제 안에서 동일 부족 영양소가 여러 성분에서 중복 매칭되면 부족 영양소를 1회만 노출한다. 단, 부족해지는 근거에는 해당 영양소와 연결된 관련 성분을 함께 표시한다.
  • 약물 간 중복(확정): 서로 다른 처방 약물이 같은 부족 영양소를 유발하는 경우에는 약물별로 반복 노출한다. 약물마다 부족해지는 이유·근거가 다르고, 약사가 "어느 약 때문인지"를 약물 단위로 확인해야 하므로 환자 기준으로 합치지 않는다.
  • UI: 하나의 영양소 블록으로 묶고 내부에 관련 성분 을 표시한다.
예시 — 부족 영양소: 마그네슘 / 관련 성분: 성분 A, 성분 B / 부족해지는 근거: · 성분 A 기준 근거 · 성분 B 기준 근거

8. 삭제 정책 재정리

확정 — 삭제는 soft delete로 통일한다(물리 삭제 없음). 부족 영양소·성분 모두 동일 방식이며, 일시 제외는 삭제가 아닌 노출 OFF를 사용한다.
  • 부족 영양소 삭제는 soft delete로 처리한다. 삭제 시 슈퍼어드민·약사웹·POS AI mate 전 서비스에서 미노출되고 원천 데이터는 보존된다.
  • 삭제 전 확인 모달: 제목 부족 영양소를 삭제하시겠습니까? · 안내(연결된 굿팜 추천 상품·관련 카테고리·관련 약물 매칭도 함께 미노출) · 버튼 취소 / 확인.
  • 삭제 조건 제한 없음: 노출 ON/OFF·약국 설정 수와 무관하게 삭제할 수 있다(원천 보존이므로). 약국 설정 수는 삭제 판정 게이트가 아니라 운영 참고 지표다.
  • 권장 운영: 일시적으로 추천에서 제외할 때는 삭제가 아니라 AI mate 노출 OFF 를 사용한다.

성분값 삭제 (성분 관리) — soft delete

슈퍼어드민 AI mate 성분 관리에서 성분값을 삭제하면 해당 성분은 전체 서비스에서 미노출 처리한다.

  • 적용 범위(모두 미노출): ① 슈퍼어드민 성분 관리 목록 · ② 슈퍼어드민 추천 관리 > 관련 약물 목록 · ③ 약사 어드민웹 관련 성분/약물 정보 · ④ POS AI mate 드럭머거/추천 상품 영역.
  • POS 처리: 처방약에 삭제된 성분코드가 포함되면 해당 성분 기준 추천·근거는 노출하지 않는다. 복합제는 삭제되지 않은 다른 성분 기준 추천은 정상 노출한다.
  • 삭제 방식: 물리 삭제가 아니라 soft delete(is_deleted / deleted_at / deleted_by / delete_reason). 화면에서는 삭제된 것처럼 보이되 DB에는 원천 데이터와 삭제 이력을 보존한다.
  • 삭제 전 확인 모달: 제목 성분값을 삭제하시겠습니까? · 안내(슈퍼어드민·약사웹·POS 미노출, 연결 부족 영양소·이유·근거 함께 미노출) · 취소 / 확인.
  • 삭제 이력: 삭제일시 · 삭제자 · 성분코드 · 성분명 · 연결 부족 영양소 수 저장.
  • 주의: 성분 삭제는 전체 어드민과 POS AI mate에 공통 반영된다. 일시적으로 추천에서 제외하려면 삭제가 아니라 노출 OFF를 사용한다.

약국 설정 수 (운영 참고 지표)

  • 표시 예: 우리약국 추천 상품을 등록한 약국 수: N곳
  • 약국 설정 수는 삭제 가능/불가 판정 게이트가 아니라 운영 참고 지표로만 사용한다. 삭제는 soft delete(원천 보존)이므로 설정 수와 무관하게 수행할 수 있다.
  • 다만 설정 약국이 많은 항목은 삭제보다 노출 OFF가 적절하므로, 운영자가 판단할 수 있도록 상세 화면에 표시한다.

9. 약사웹 OFF 상태 정책

슈퍼어드민에서 특정 부족 영양소를 OFF로 변경해도 약사웹에서 완전히 숨기지 않는다.

구분처리
POS AI mate미노출
약사웹 목록AI mate 노출 중지 상태로 표시(숨기지 않음)
약사웹 상세조회 + 등록/수정/삭제 허용(사전 세팅)
신규 등록/수정/삭제허용 — 노출 재개 전에 미리 준비할 수 있게 한다
기존 설정값보존(삭제하지 않음)
다시 ON 변경저장된 설정값 그대로 노출
  • 핵심 정의: OFF는 데이터 삭제가 아니라 POS AI mate 추천 노출 중지 상태다. 약사가 "설정이 사라졌다"고 오해하지 않도록 숨기지 않고 노출 중지 상태로 보여준다.
  • 편집 허용(확정): OFF 상태에서도 약국이 우리약국 추천 상품을 미리 등록·수정할 수 있다. 저장한 값은 POS에 즉시 노출되지 않고, 노출이 재개되면 그대로 사용된다. (노출 재개 시점에 급히 세팅해야 하는 부담을 없애기 위함)
  • 약사웹 안내 문구: 현재 굿팜 정책에 따라 이 부족 영양소의 POS AI mate 추천 노출이 중지되었습니다. 다만 우리약국 추천 상품은 지금도 등록·수정할 수 있으며, 노출이 다시 시작되면 저장한 설정값이 그대로 사용됩니다.
  • 하단 안내: AI mate 노출 중지 상태 — 지금 등록·수정한 설정값은 노출이 재개되면 그대로 사용됩니다. (POS에는 아직 노출되지 않음)

10. 화면별 반영 (현재 프로토타입 기준)

슈퍼어드민 — AI mate 추천 관리 (부족 영양소 기준)

  • 목록: 부족 영양소 · 관련 카테고리 · 관련 약물 수 · 노출(ON/OFF) · 최종 수정일. 행 클릭 → 상세 페이지.
  • 상세: AI mate 노출 토글(상단) · 관련 카테고리 · 굿팜 추천 상품(최대 2) · 관련 약물 목록(성분명 클릭 → 성분 관리 이동) · 수정 이력. 저장/삭제는 우측 하단.

슈퍼어드민 — AI mate 성분 관리 (성분코드 기준)

  • 목록: 성분코드 · 성분명 · 연결 부족 영양소 수 · 최종 수정일. 검색(성분코드·성분명).
  • 상세: 성분명(공통값) 수정 → 확인 모달(전 화면 공통 반영) · 연결 부족 영양소 목록(부족해지는 이유·근거를 조합별 수정, 부족 영양소 연결 추가/삭제) · 수정 이력.

약사웹 — AI mate 추천 상품 설정

  • 목록 8열: 부족 영양소 · 관련 카테고리 · 관련 약물 수 · 우리약국 추천 상품 · 굿팜 추천 상품 · AI mate 노출 상태(ⓘ 툴팁) · 최종 수정. 행 클릭 → 상세 페이지.
  • 상세: 기본 정보 · 관련 약물(조회) · 추천 상품 설정(우리약국 최대 3 등록/정렬/삭제 · 굿팜 조회) · 저장은 우측 하단. 상품 표기는 바코드 대신 제조사(매입처별 바코드 상이).
  • OFF 시: 상단 안내 배너 + 편집 버튼 비활성화(툴팁).

POS — 굿팜 AI MATE (시안 D 확정)

  • 레이아웃 확정: 시안 D(처방 약물표 + 드럭머거 2열)로 확정한다. 시안 C(약물 카드형)는 화면에서 숨김(코드는 보존, 시안 전환 토글 미노출).
  • 처방 약물표 하단 드럭머거 & 추천제품. 부족 영양소 블록 = 좌(부족 영양소·관련 카테고리·부족해지는 근거·참고문헌 ⓘ) / 우(우리약국·굿팜 2열 비교). 좌/우, 우리약국/굿팜 사이에 구분선을 둔다.
  • 섹션 구분·귀속: 추천 영역은 옅은 배경과 좌측 액센트로 처방 약물 행과 구분하되 위쪽은 붙여 해당 약물에 종속됨을 표현한다. 섹션 좌측 상단에 Drug Mugger 로고(시안 B · 첫 글자만 대문자) 와 함께 <약물명> 복용 시 부족해질 수 있는 영양소 문구를 두어 어느 처방 약물 기준인지 명시한다.
  • UI 문구·표기: 열 라벨 우리약국 추천 / 굿팜 추천(라벨 텍스트만 최소 컬러, 상품 카드·버튼·번호는 중립 톤·배경 없음). 우리약국 미등록 시 등록 버튼 ＋ 추천 상품 등록.
  • 개수 표기: 라벨 옆 총 개수 배지는 표기하지 않는다. 우리약국 상품이 2개 이상일 때 좌측 우선순위 번호(1·2·3) 로만 순서·개수를 인지시킨다.
  • 다약제(처방 약물 10개 이상) 정책 확정 필요: 처방 약물 수에 대한 제한·접기 규칙은 현재 없다(접기는 약물별 부족 영양소 5개 초과에만 적용). 다약제 처방에서 세로 길이가 길어지므로 ①현행 유지(전체 나열) ②추천 섹션 기본 접기(약물 클릭 시 펼침) ③상위 N개 약물만 노출 + 더보기 중 확정한다.
  • 추천 상품이 하나라도 있는 부족 영양소만 노출(둘 다 없으면 블록·"추천 상품 없음" 모두 미노출).
  • 부족 영양소가 5개를 초과하면 상위 5개 + +N개 더보기(DIND 분포상 완제품의 82%가 영양소 5개 이하). 우리약국 2개 이상이면 우선순위 번호. 긴 상품명은 2줄 말줄임 + hover 전체명.
