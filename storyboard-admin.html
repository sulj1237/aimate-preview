부족 영양소 관리 · 성분 관리

부족 영양소 기준(v2)으로 부족 영양소 관리와 성분 관리(성분코드)를 운영한다. 3층 grain — 부족 영양소는 부족 영양소 관리에서, 성분명(성분코드 공통값)과 부족해지는 이유·근거(성분코드×부족영양소 조합값)는 성분 관리에서 편집한다. 노출은 ON/OFF, 제거는 soft delete(원천 보존·전 서비스 미노출)로 처리한다.

디스크립션 (영역·기능·인터랙션)
#영역/항목기능 및 정책인터랙션 / 처리·예외
1AI mate 노출 토글상단에서 이 부족 영양소의 노출 ON/OFF 제어. OFF=POS 미노출, 약사웹 노출 중지 표시.토글 즉시 반영, 저장 시 확정. 목록의 노출 배지와 연동.
2기본정보부족 영양소·관련 약물 수·관련 카테고리(영양소 종속 값) 조회.부족 영양소명 자체는 조합 기준값 → 여기서는 참조.
3굿팜 추천 상품(최대 2)상담 참고용 추천 상품 최대 2개 등록. 굿팜 보유 상품 검색으로 추가·순서 조정·삭제.2개 초과 추가 시 기본 추천 상품은 최대 2개까지 등록할 수 있습니다. · 중복 시 이미 추가된 상품입니다.
4관련 약물 목록이 영양소를 고갈시키는 관련 약물(성분코드·성분명·이유·근거). 조회 전용.성분명 클릭 → AI mate 성분 관리 상세로 이동해 성분명·이유·근거 수정. 삭제된 성분은 미노출.
5override 상태원천 대비 운영 수정 상태값: 원천값 사용/운영 수정값 사용/원천 변경 있음/검토 필요.운영 수정 중이면 운영 수정 적용 중 배지, 원천 변경 시 원천 변경 있음 알림.
6결측 표기이유/근거 미입력 조합은 근거 미등록/이유 미등록/검토 필요로 표기.결측이면 노출 전 검토 유도.
7수정 이력필드·전/후·수정일시·수정자 기록.감사·롤백 근거.
8저장 / 삭제우측 하단. 삭제는 soft delete(전 서비스 미노출·원천 보존).삭제 클릭 → 확인 모달(취소/확인) → Case(삭제 확인 모달) 참고.

CASES부족 영양소 관리 · 케이스별 화면— 목록 ON/OFF 공존 · 데이터 0 · soft delete 반영 · 상세 OFF · 삭제 확인 모달 · 데이터 조건별
Case 1목록 — ON/OFF 행 공존기본
▶ superadmin.html · 노출 ON(비타민B12·코엔자임Q10·칼슘·마그네슘)과 OFF(비타민D·엽산·칼륨)가 한 목록에 공존.
  • 달라지는 점 노출 배지가 ON/OFF로 섞여 표시 — 노출 여부와 무관하게 Master는 목록에 유지된다(노출 OFF ≠ 삭제).
  • 정책 근거 노출은 운영 토글, 제거는 soft delete로 분리. OFF는 언제든 다시 ON 가능.
  • 인터랙션 행 클릭 → 상세 이동. 노출 상태 변경은 상세 토글에서.
Case 2목록 — 데이터 0건빈 상태
  • 달라지는 점 검색 결과나 등록 Master가 없을 때 표 본문에 데이터가 없습니다. 한 줄만 노출.
  • 정책 근거 목록0 표준 문구(부족 영양소 관리). 성분 관리 이력0은 데이터가 존재하지 않습니다.로 구분.
  • 인터랙션 검색 조건 해제 또는 ＋ DIND 추가로 데이터 생성.
Case 3목록 — soft delete 반영삭제 반영
▶ superadmin.html?case=deleted · 오메프라졸(A02BC01) 성분 삭제 처리 → 관련 약물 수에서 빠지고 관련 약물 목록·매칭에서 사라짐.
  • 달라지는 점 삭제된 성분과 매칭된 부족 영양소(비타민B12·칼슘·마그네슘)의 관련 약물 수가 감소. 부족 영양소 자체 삭제 시엔 목록 행이 사라짐.
  • 정책 근거 soft delete — 원천 보존, 전 서비스 미노출. 성분 삭제는 관련 약물 집계·목록에서 제외.
  • 인터랙션 관련 약물 수 클릭 없이 즉시 반영. 삭제 이력은 성분 관리에서 확인.
Case 4추천 상세 — 노출 OFFOFF
▶ superadmin.html?master=4 · 비타민 D(노출 OFF) 상세. 상단 토글이 OFF 상태.
  • 달라지는 점 노출 토글이 OFF — POS AI mate 미노출, 약사웹에는 노출 중지 표시. 데이터는 그대로 편집 가능.
  • 정책 근거 노출 OFF는 일시 중지 수단(삭제 아님). 검토 필요/결측 항목은 OFF로 두고 보완 후 ON.
  • 인터랙션 토글 ON → 저장 시 전 서비스 노출 반영.
Case 5부족 영양소 삭제 확인 모달soft delete
▶ superadmin.html?master=4&modal=reco-del · 부족 영양소를 삭제하시겠습니까? 확인 모달(취소/확인).
  • 달라지는 점 상세 우측 하단 삭제 → 확인 모달. 안내: 슈퍼어드민·약사 어드민웹·POS AI mate 모두 미노출, 연결된 굿팜 추천 상품·관련 카테고리·관련 약물 매칭도 함께 미노출.
  • 정책 근거 soft delete — 원천 데이터 보존, 화면에서만 미노출. 삭제 후 목록·관련 약물에서 사라짐.
  • 인터랙션 [취소]=닫기 / [확인]=삭제 실행 후 목록 복귀.
Case 6데이터 조건 — 추천 상품 2 도달 · override · 결측데이터
  • 2 도달 굿팜 추천 상품이 이미 2개면 검색 추가 차단 → 기본 추천 상품은 최대 2개까지 등록할 수 있습니다. 삭제로 슬롯을 비운 뒤 재추가.
  • override 운영 수정이 원천과 다르면 운영 수정 적용 중 배지 · 원천이 이후 변경되면 원천 변경 있음으로 재검토 유도. 상태값=원천값 사용/운영 수정값 사용/원천 변경 있음/검토 필요.
  • 결측 조합별 이유/근거 미입력 시 근거 미등록/이유 미등록/검토 필요 표기. 결측 항목은 노출 전 보완 권장.

3성분명성분코드 공통값. 상세에서 수정 시 전 화면 공통 반영.수정은 상세 → 확인 모달 경유.
4연결 부족 영양소 수이 성분코드와 매칭된 부족 영양소 개수(조합 수). 삭제된 부족 영양소 제외.0이면 연결 없음.
5최종 수정일공통값/조합값 마지막 수정일.정렬·추적 기준.
6＋ DIND 추가성분×부족영양소 매칭 단건 등록 진입점.모달로 진입 → Case(DIND 추가) 참고.
7목록 정렬기본 정렬 ① 최종 수정일 최신 → ② 연결 부족 영양소 수 많은 순.진입 시 자동 정렬.

Case 1성분 상세 — 공통값·조합값 편집상세
▶ superadmin.html?ing=A02BC01 · 오메프라졸. 성분명(공통값) / 연결 부족영양소 목록(조합별 이유·근거) / 연결 추가·삭제 / 수정 이력.
  • 달라지는 점 성분명은 성분코드 공통값(한 번 수정 → 연결된 모든 조합 반영). 연결 부족영양소마다 부족해지는 이유·근거를 개별(조합값) 편집.
  • 정책 근거 3층 grain — 공통값=성분 관리 상단, 조합값=연결목록. 편집 위치가 축으로 갈림.
  • 인터랙션 ＋ 부족 영양소 추가로 연결 확장, 각 행 삭제로 연결 해제. 성분명 수정 시 확인 모달(Case 6-데이터).
Case 2DIND 추가 모달검토 필요·OFF
▶ superadmin.html?modal=dind · 성분코드×부족영양소 매칭 단건 등록. 성분코드·성분명·부족 영양소·이유·근거 입력.
  • 달라지는 점 새 DIND(약물-영양소 상호작용) 매칭을 수동 등록. 기존 성분코드면 성분명 자동 사용, 기존 조합이면 연결만 추가.
  • 입력 참고 성분코드 입력 시 앞부분이 같은 기존 성분코드가 있으면 성분명·연결 부족 영양소·이유·근거를 참고 영역에 표시(전체 일치 코드는 전체 일치 배지). 참고 표시일 뿐 값은 각 항목을 직접 입력하며, 매칭은 성분코드 전체 일치일 때만 유효. (신규) 개발 확인 필요
  • 정책 근거 등록 DIND는 검토 필요·기본 노출 OFF(AI mate·약사웹 미노출)로 진입 → 검토 후 부족 영양소 관리에서 노출 ON. 일괄 업로드·재업로드 병합 TBU
  • 인터랙션 [취소]/[추가]. 추가 후 성분·부족 영양소 관리 양쪽 연결 반영.
Case 3성분 삭제 확인 모달soft delete
▶ superadmin.html?ing=A02BC01&modal=ing-del · 성분값을 삭제하시겠습니까? 확인 모달(취소/확인).
  • 달라지는 점 성분 상세 하단 삭제 → 확인 모달. 안내: 슈퍼어드민·약사 어드민웹·POS AI mate 모두 미노출, 연결된 부족 영양소·부족해지는 이유·근거도 함께 미노출.
  • 정책 근거 성분 soft delete — 전 서비스 미노출·원천/이력 보존. POS는 삭제 성분 기준만 미노출(복합제 나머지 정상). 일시 제외는 삭제 아닌 노출 OFF.
  • 인터랙션 [취소]/[확인]. 확인 시 목록·관련 약물에서 제거, 삭제 이력에 기록.
Case 4성분 관리 — soft delete 반영삭제 반영
▶ superadmin.html?page=ingr&case=deleted · 오메프라졸(A02BC01) 삭제 처리되어 성분 목록에서 사라짐.
  • 달라지는 점 삭제 성분이 성분 목록에서 제거되고, 부족 영양소 관리 관련 약물·약사웹·POS에서도 공통 미노출. 삭제 이력에는 남음.
  • 정책 근거 적용범위=슈퍼어드민 성분관리·추천관리 관련약물·약사웹·POS 모두 미노출. POS 복합제는 나머지 성분 정상 노출.
  • 인터랙션 삭제 이력 모달에서 확인 가능.
Case 5데이터 조건 — 성분명 공통 반영 · 연결 추가/삭제데이터
  • 성분명 수정 성분명(공통값) 변경 → 확인 모달: 이 성분과 연결된 모든 부족 영양소 항목 및 AI mate 화면에 공통 반영. 영향 범위(연결 부족 영양소·관련 노출 대상) 표기 후 [저장].
  • 연결 추가 ＋ 부족 영양소 추가로 조합 신설 — 새 조합은 이유·근거 이유 미등록/근거 미등록·검토 필요로 진입.
  • 연결 삭제 특정 부족 영양소 연결만 해제(성분 자체 삭제 아님). 이력에 항목·전/후 기록. 성분 전체 제거는 성분 삭제(soft delete) 사용.
