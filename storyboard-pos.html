처방 상세 분석 · 드럭머거 & 추천제품 (시안 D · 2열)

POS 약사가 처방을 조회하면 처방 약물표 하단에 부족해질 수 있는 영양소와 추천 제품이 자동으로 노출된다. 좌측은 영양소·근거, 우측은 우리약국(우선강조) vs 굿팜(보조) 2열 비교. POS는 조회 전용이며, 신규 개발 범위는 이 추천 영역에 한정된다(그 외 화면은 as-is).

1처방 약물표 (as-is)기존 처방 상세 분석의 약물 목록. 신규 범위 아님.표 하단에 추천 영역이 자동으로 이어 붙어 노출된다.
2영양소 상담 추천 영역신규약물표 하단 자동 노출. 처방 약물의 드럭머거 분석 결과(부족 영양소)만 대상. §4 정규화로 표준 영양소명만 표기.노출/상품클릭/CTA클릭 로그 적재. 조회 전용(POS 편집 불가).
3노출 레이아웃시안 D 확정시안 D(표 + 드럭머거 2열)로 확정. 시안 C(약물 카드형)는 화면에서 숨김(코드는 보존).탐색용 시안 토글 미노출 — 항상 시안 D로 렌더.
4부족 영양소 블록 (좌열)신규부족영양소명 + 관련 카테고리 배지 + 부족해지는 근거 문구 + 참고문헌 ⓘ. §7 결측 시 ⓘ만 미노출.ⓘ hover → 참고문헌 tooltip. 근거는 §3 override 최종 반영값만(원천/수정 구분 미노출).
5우리약국 컬럼 (우측·우선강조)신규우리약국 등록 제품을 우선 강조. 최대 3개. 2개 이상이면 우선순위 번호(1·2·3) 표기. 라벨 옆 총 개수 배지는 미표기(번호로만 구분).상품명 hover → 전체명 tooltip. 제품 클릭 로그.
6굿팜 컬럼 (우측·보조)신규굿팜 추천 제품(보조). 최대 2개. 우리약국과 나란히 2열 비교.우리약국·굿팜 중 하나라도 있으면 블록 노출(§7,§10 hide-when-no-products).
7우리약국 미등록 CTA신규해당 영양소에 우리약국 제품이 없을 때 ＋ 추천 상품 등록 노출(굿팜만 있는 경우).클릭 → 약사웹 설정 새 창(약국ID·성분코드·영양소ID 전달). 실패 시 설정 페이지를 열 수 없습니다. 잠시 후 다시 시도해 주세요.
8복합제 통합 블록신규§6: 동일 영양소가 복수 성분에서 중복 매칭되면 1블록으로 통합. 근거에 관련 성분 병기(성분 A, 성분 B / 근거 · 성분A기준 · 성분B기준).제품 목록은 통합, 근거만 성분별로 구분 표기.

Case 4한 약물 · 4영양소 혼합혼합

Case 6노출 OFF → 영역 미노출미노출
▶ pos-aimate.html?case=off · 추천 영역이 비어 있는 게 정상
  • 달라지는 점 처방 약물표만 보이고 추천 영역이 통째로 안 나온다(임베드가 비어 보이는 게 정상 동작).
  • 정책 근거 §9 노출 OFF 영양소는 POS 미노출. 대상 영양소가 전부 OFF면 영역 자체가 사라진다.
  • 인터랙션 안내 문구도 표시하지 않음(문구 없이 미노출). 노출 로그도 발생하지 않음.
Case 7매칭 없음 → 추천 없음미노출
▶ pos-aimate.html?case=no-product · 매칭 제품 0 → 영역 미노출
  • 달라지는 점 부족 영양소는 있어도 우리약국·굿팜 어디에도 매칭 제품이 없어 추천 영역이 나오지 않는다(비어 보이는 게 정상).
  • 정책 근거 상품 0개 = 영역 미노출(문구 없음). 매칭 0도 동일 처리(§7,§10).
  • 인터랙션 별도 빈 상태 문구/CTA 없음. 제품 등록 시 다음 조회부터 자동 노출.
Case 8복합제 통합데이터 케이스
  • 상황 한 처방의 복합제에서 성분 A·성분 B가 동일 영양소(예: 마그네슘)를 각각 소모 → 같은 영양소가 중복 매칭됨.
  • 화면 처리 §6 통합: 영양소당 1블록으로 합치고 관련 성분: 성분 A, 성분 B 병기. 근거는 · 성분A기준 · 성분B기준으로 성분별 구분. 제품 목록은 통합해 한 번만 노출.
  • 주의 블록을 성분 수만큼 중복 노출하지 말 것. 근거만 성분별로 나열, 영양소·제품은 1회.
Case 9삭제 성분 포함 복합제데이터 케이스
  • 상황 복합제 성분 중 하나가 soft delete(§8)된 성분코드. 나머지 성분은 정상.
  • 화면 처리 삭제 성분 기준의 추천·근거만 미노출하고, 남은 성분 기준 추천·근거는 정상 노출(부분 노출). 통합 블록의 근거 목록에서 삭제 성분 항목만 빠진다.
  • 주의 삭제 성분 하나 때문에 블록 전체를 숨기지 말 것. 남은 성분 매칭이 있으면 반드시 노출.
Case 10결측 — 관련 카테고리·부족 이유·근거 없음결측
▶ pos-aimate.html?case=missing · 관련 카테고리·참고문헌·부족 이유가 없어도 추천 상품(필수 값)이 있으면 블록 노출(없는 항목만 생략)
  • 관련 카테고리 없음 카테고리 배지를 미표기(빈 배지·- 노출 안 함). 영양소명만 표시. (§7)
  • 참고문헌 없음 부족 이유 문구·상품은 정상 노출, 참고문헌 ⓘ 아이콘만 미표기. (§7)
  • 부족 이유 없음 부족 이유가 없어도 추천 상품(필수 값)이 있으면 블록은 노출되고, 부족해지는 근거 문구만 생략된다(데모의 마그네슘). (§7)
  • 주의 블록은 추천 상품(필수 값)이 없을 때만(데이터 부재) 미노출. 카테고리 없음=배지만, 참고문헌 없음=ⓘ만, 이유 없음=근거 줄만 생략(블록은 노출).
