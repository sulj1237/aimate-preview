표기 — 미확정 항목은 TBU(빨강). 본문은 02 정책(v2) 을 화면 단위로 풀어 쓴 것이다. 이 문서는 기획자 계정에서 상단 ✏ 수정으로 직접 편집할 수 있다.

P1. POS 굿팜 AI MATE

신규 범위는 처방 약물표 하단 드럭머거 &amp; 추천제품 영역뿐이다(그 외 as-is).

AIM-C-01 · 드럭머거 & 추천제품 (시안 D · 2열)

화면 개요

  • 무엇: 처방 약물별로 부족해지는 영양소 + 추천 상품을 상담 참고로 노출.
  • 진입: 처방 상세 분석 → 처방 약물표 하단 자동 노출.
  • 조회 흐름(정책 §2): 처방약 성분코드 → DIND 상세 성분코드(전체 일치 매칭) → 부족 영양소 → (영양소 기준) 추천 상품 → (성분코드×영양소) 근거.

항목별 상세

  • 섹션 구분·귀속: 추천 영역은 옅은 배경 + 좌측 액센트로 처방 약물 행과 구분하되 위쪽은 붙여 해당 약물에 종속됨을 표현. 섹션 좌측 상단에 Drug Mugger 로고(시안 B) + <약물명> 복용 시 부족해질 수 있는 영양소 문구.
  • 부족 영양소 블록 = 좌우 2열
  • 좌: 부족 영양소명 · 관련 카테고리(배지) · 부족해지는 근거 · 참고문헌 ⓘ(hover).
  • 우: 우리약국 추천 상품(우선 강조) | 굿팜 추천 상품(보조) 2컬럼 비교.
  • 노출 정책(정책 §7): 우리약국/굿팜 중 하나라도 있는 부족 영양소만 노출. 둘 다 없으면 블록·"추천 상품 없음" 모두 미노출.
  • 결측: 추천 상품(필수 값)이 있으면 블록 노출하고 없는 항목만 생략(이유 없음→근거 문구 생략, 참고문헌 없음→ⓘ 생략, 카테고리 없음→배지 생략). 추천 상품이 없을 때만(데이터 부재) 블록 미노출.
  • 복합제(정책 §6): 동일 영양소가 여러 성분에서 중복되면 1블록으로 통합하고 내부에 관련 성분을 함께 표기.
  • 긴 상품명: 카드 2줄 말줄임 + hover 전체명 tooltip. 우리약국 2개 이상이면 우선순위 번호(1·2·3).
  • 5개 초과: 상위 5개 노출 + +N개 더보기.
  • 우리약국 미등록: 우측에 ＋ 추천 상품 등록 CTA(약사웹 이동).

P2. 약국 어드민 · 약사웹

▶ pharmacy-web.html · GP 디자인 시스템(라이트·민트·Pretendard).

AIM-P-01 · 추천 상품 설정 — 목록

  • 컬럼(8열): 부족 영양소 · 관련 카테고리 · 관련 약물 수 · 우리약국 추천 상품 · 굿팜 추천 상품 · AI mate 노출 상태(ⓘ 툴팁) · 최종 수정.
  • 노출 상태: ON AI mate 노출(그린) / OFF AI mate 노출 중지(옐로) — OFF도 숨기지 않는다(정책 §9).
  • 행 클릭 → 상세 페이지로 전환(모달 아님).

AIM-P-02 · 추천 상품 설정 — 상세(페이지)

  • 상단: ‹ 뒤로가기 + 부족 영양소명. 저장은 우측 하단.
  • 기본 정보: 부족 영양소 · 관련 카테고리 · 관련 약물 수.
  • 관련 약물(조회 전용): 성분명 · 부족해지는 이유 · 근거(참고문헌).
  • 추천 상품 설정: 우리약국 추천 상품(최대 3 · 등록/정렬/삭제 · 목록 하단에 추가 버튼) → 아래 굿팜 추천 상품(조회). 상품 표기는 바코드 대신 제조사(매입처별 바코드 상이).
  • OFF 시: 상단 안내 배너 노출. 편집은 허용(사전 등록·수정·삭제·정렬·저장 모두 가능) — 저장값은 POS에 즉시 노출되지 않고 노출 재개 시 그대로 사용. 하단 안내 AI mate 노출 중지 상태 — 지금 등록·수정한 설정값은 노출이 재개되면 그대로 사용됩니다. (POS에는 아직 노출되지 않음).

AIM-P-03 · 굿팜 보유 상품 검색 모달

  • 굿팜 보유 상품에서 검색·선택만 허용(약국 로컬/POS 아님 — POS 미사용 약국 대응). 검색 키는 상품명·바코드, 결과 표기는 제조사. 결과 없음/중복/최대 도달 안내.

P3. 슈퍼어드민

▶ superadmin.html · 메뉴 AI Mate 추천 상품 관리 하위에 추천 상품 관리 / 성분 관리 2개.

AIM-S-01 · AI mate 추천 관리 — 목록 (부족 영양소)

  • 컬럼: 부족 영양소 · 관련 카테고리 · 관련 약물 수 · 노출(ON/OFF) · 최종 수정일. 행 클릭 → 상세 페이지.

AIM-S-02 · AI mate 추천 관리 — 상세(페이지)

  • AI mate 노출 토글(상단) → 기본 정보(부족 영양소·관련 약물 수·관련 카테고리 편집) → 굿팜 추천 상품(최대 2 · 검색 추가/순서/삭제) → 관련 약물 목록(성분명 클릭 → 성분 관리 이동) → 수정 이력. 저장/삭제는 우측 하단.
  • 삭제 정책(정책 §8): soft delete — 확인 모달(부족 영양소를 삭제하시겠습니까? · 취소/확인) 후 슈퍼어드민·약사웹·POS 전 서비스 미노출, 원천 데이터는 보존. 일시 제외는 삭제가 아닌 노출 OFF 사용.

AIM-S-03 · AI mate 성분 관리 — 목록/상세 (성분코드)

  • 목록: 성분코드 · 성분명 · 연결 부족 영양소 수 · 최종 수정일. 검색(성분코드·성분명).
  • 상세(모달): 성분명(공통값) 수정 → 확인 모달(전 화면 공통 반영) · 연결 부족 영양소 목록(부족해지는 이유·근거를 (성분×영양소) 조합별 수정, 부족 영양소 연결 추가/삭제) · 수정 이력.
  • 핵심: 성분명은 성분코드 기준 공통값(한 번 수정 → 전 화면 반영), 이유·근거는 (성분코드 × 부족 영양소) 조합별 독립.
  • 매칭 기준(정책 §2): 성분코드 전체 일치로 매칭·관리한다. 성분명이 같아도 코드가 다르면 별개 레코드.
  • DIND 추가 입력 참고: 입력한 성분코드의 앞 4자리가 같은 기존 성분코드가 있으면 성분명·연결 부족 영양소·이유·근거를 모달 안에 참고용으로 표시한다(자동 복사·병합 없음).
참고 — 슈퍼어드민에서 DIND 단건 수동 추가(성분코드 × 부족 영양소 매칭)가 가능하다. 추가된 항목은 검토 필요 · 노출 OFF로 등록되며, 검토 후 노출 ON으로 전환한다. 단, DIND 일괄 업로드와 원천 재업로드 시 override 병합 UX는 정책 §3와 함께 상세 정책 확정 후 반영한다.
